[Инструкция по выбору VPS и настройке домена.](vps.md)

**Slipgate** - Единый менеджер туннелей для серверов Linux. Управляет DNS-туннелями (DNSTT, NoizDNS, Slipstream, VayDNS) и HTTPS-прокси (NaiveProxy) через сервисы systemd, поддерживает маршрутизацию DNS через несколько туннелей и управление пользователями. Предназначен для использования с Android VPN-приложением [SlipNet](https://github.com/anonvector/SlipNet).

*Возможности:*

- **Поддержка нескольких транспортов**: DNSTT/NoizDNS (DNS-туннели с шифрованием Curve25519), Slipstream (DNS на базе QUIC), VayDNS (DNS на базе KCP с Curve25519), NaiveProxy (HTTPS через Caddy), StunTLS (SSH через TLS + WebSocket)

- **Два варианта бэкенда**: встроенный SOCKS5-прокси или перенаправление через SSH (поддерживается пользовательский SSH-порт)

- **Маршрутизация DNS**: режим одного туннеля или нескольких туннелей с распределением запросов по доменам

- **Внешняя маршрутизация**: перенаправление DNS-запросов для домена на пользовательский порт для протоколов, управляемых пользователем

- **Интеграция с WARP**: дополнительная маршрутизация исходящего трафика через Cloudflare WARP (альтернативный подход см. в [dnstun-ezpz](https://github.com/aleskxyz/dnstun-ezpz))

- **Управление пользователями**: поддержка нескольких пользователей SSH + SOCKS с одновременной аутентификацией всех пользователей; пакетное создание до 500 пользователей за один вызов

- **Панель мониторинга в реальном времени**: интерактивный TUI-интерфейс с отображением загрузки CPU, использования RAM, графиков трафика, статистики подключений по каждому протоколу и состояния туннелей

- **Диагностика**: встроенные проверки работоспособности сервисов, портов, ключей, DNS-разрешения и автозапуска после перезагрузки

- **Интерактивный TUI и CLI**: настройка через меню или управление с помощью скриптов и подкоманд

- **Интеграция с systemd**: создание сервисов, управление их жизненным циклом и просмотр журналов

- **Автоматический TLS**: получение сертификатов Let's Encrypt через Caddy для туннелей NaiveProxy

- **Самообновление**: проверка новых версий и замена бинарных файлов из релизов GitHub

- **Передача конфигурации клиенту**: генерация URI вида `slipnet://` для импорта настроек в приложение одним нажатием

[Ссылка на репозиторий](https://github.com/anonvector/slipgate)



## Установка на сервере

**Внимание! Если вы уже разворачивали схожие проекты на своем сервере (например, [MasterDnsVPN](masterdnsvpn.md)), остановите сервис командой** `systemctl stop masterdnsvpn.service` **во избежание проблем.**

### slipstream

Разработчик подготовил bash скрипт для интерактивной установки. 

Выполните эту команду в терминале:

```bash
curl -fsSL https://raw.githubusercontent.com/anonvector/slipgate/main/install.sh | sudo bash
```

Вас встретит предложение выбрать транспорт:

```bash
   _____ _ _       _____       _
  / ____| (_)     / ____|     | |
 | (___ | |_ _ __| |  __  __ _| |_ ___
  \___ \| | | '_ \ | |_ |/ _` | __/ _ \
  ____) | | | |_) | |__| | (_| | ||  __/
 |_____/|_|_| .__/ \_____|\__,_|\__\___|
             | |
             |_|

[+] Downloading slipgate (linux/amd64)...
[+] Running slipgate install...

  Which transports do you want to install?

  Transports:
    1) DNSTT / NoizDNS — DNS tunnel
    2) Slipstream — QUIC DNS tunnel
    3) VayDNS — KCP DNS tunnel
    4) NaiveProxy — HTTPS proxy with Caddy
    5) StunTLS — SSH over TLS + WebSocket proxy
    6) SSH — Direct SSH tunnel
    7) SOCKS5 — Direct SOCKS5 proxy
    8) All
  Choice (comma-separated, e.g. 1,3,4): 
```

Т.к. мы делаем slipstream, указываем цифру 2.

Выберите SOCKS5 бэкэнд в следующем пункте.

Затем, скрипт спросит Ваш домен:

```bash
  ── slipstream ──
  Domain for slipstream (e.g. s.example.com):
```

Здесь мы указываем NS запись, которую мы сделали ранее (например, `v.example.dpdns.org`)

Затем, создайте пользователя:

```bash
  ── User Setup ──────────────────────────────────────

  Create a user now? [Y/n]: y
  Username [user1]: gaming
  Password (leave blank to generate): gaming
[+] User "gaming" created (SSH + SOCKS)
```

После этого, скрипт предлагает нам сделать WARP эндпоинтом. Если Вы желаете спрятать IP адрес сервера за WARP, согласитесь. Скрипт скачает wireguard, создаст WARP аккаунт и затем сгенерирует конфиг.

 В конце работы скрипта вы увидите следующее:

```bash
  ══════════════════════════════════════════════════════
    Installation Summary
  ══════════════════════════════════════════════════════

    Transports: 1 installed
    Tunnel    : slipstream (backend: socks)

    DNS Records Required:
      A  record: ns.example.dpdns.org → your server IP
      NS record: v.example.dpdns.org → ns.example.dpdns.org

    Client Configs:

    [slipstream] gaming
    slipnet://закодированная в base64 ссылка, которая содержит вид транспорта, домен, логин и пароль созданного юзера

  ══════════════════════════════════════════════════════

  Next steps:
    - Set up DNS records above with your domain registrar
    - Import the slipnet:// config into the SlipNet app
    - Add more tunnels: sudo slipgate tunnel add
    - Add more users:   sudo slipgate users

[+] Done! Run 'sudo slipgate' to see the menu.
```

На этом установка завершена. Можно пробовать подключиться к серверу.

### Установка других транспортов

Выполните `slipgate` в терминале. 

Выберите пункт `7. Install (advanced)` и следуйте указаниям скрипта.

Важно уточнить, что для каждого туннеля требуется свой поддомен. Если планируете экспериментировать и/или держать несколько транспортов на одной VPS, предварительно создайте NS записи. [Подробнее в мануале на утилиту.](https://github.com/anonvector/slipgate#domain-layout)

## Настройка на клиенте

### SlipNet

Для тестов обратимся к приложению на Android: [Ссылка на репозиторий](https://github.com/anonvector/SlipNet)

Скачайте .apk файл, запустите, нажмите `+` в правом нижнем углу экрана, выберите `Import` и вставьте туда `slipnet://` ссылку. Откройте профиль.

Просканируйте резолверы встроенными в софт средствами. Для этого нажмите на кнопку `Scan for Working Resolvers`, опуститесь в самый низ, вставьте заранее подготовленные DNS резолверы, либо импортируйте файл. Затем, нажмите `Load IPs` и `Start Scan`.

Приложение будет неспешно сканировать резолверы на доступность и параллельно будет проводить E2E тесты. 

Нас интересуют резолверы, у которых **Score 6/6**. Отметьте их галочкой (slipstream поддерживает параллельную работу максимум через 8 резолверов), затем `Apply`. Выбранные резолверы будут прописаны в настройках конфига.

Теперь пробуйте выйти в интернет и замерить скорость.

## Ответы на некоторые вопросы

1. **Будет ли работать, если получился микс из резолверов, например Score 5/6 и Score 6/6?** Да, будет, но медленнее, чем могло быть.

2. **Как можно увеличить скорость?** По умолчанию QUIC отключен в настройках. Его можно включить руками, скорость должна слегка вырасти. *Были репорты, что QUIC блокируют при БС.* 

3. **500 Кбайт/с это несерьезно, хочу еще быстрее!** Используйте [MasterDnsVPN](masterdnsvpn.md) либо воспользуйтесь инструкцией [Проверка резолверов на скорость](speedtest.md)  (но софт разработан под сервер с MasterDnsVPN, ололо)

4. **Я наигрался со slipgate и хочу вернуться на MasterDnsVPN!** 
* В первую очередь остановите сервисы туннеля и dnsrouter самого slipgate. 
  
  * Для начала, остановите туннель: `slipgate > Tunnels > Stop tunnel > название туннеля`.
  
  * Затем, остановите slipgate-dnsrouter.service командой: `systemctl stop slipgate-dnsrouter.service`
  
  * Остановите смежные сервисы, требующиеся для работы slipgate: `systemctl stop slipgate-socks5.service` и `systemctl stop slipgate-warp.service`

* Запустите MasterDnsVPN: `systemctl start masterdnsvpn.service`. 
  
  * Убедитесь, что сервис работает, выполнив:`journalctl -u masterdnsvpn.service -f`  Вы должны увидеть эту строку: `06/22 10:16:36 [MasterDnsVPN Server] [INFO] 📡 UDP Listener Ready, Addr: 0.0.0.0:53, Readers: 6, Workers: 10, Queue: 32768, Sockets: 6`
