# MasterDnsVPN

*Последний раз отредактировано: 24.06.2026*

[Github проекта](https://github.com/masterking32/MasterDnsVPN/)

**MasterDnsVPN** - это продвинутый инструмент для туннелирования TCP-трафика через DNS-запросы и ответы. Он разработан для работы в сетях с сильной цензурой и блокировками, позволяя обходить ограничения там, где другие методы подключения недоступны.

*Основные особенности проекта:*

- Стабильность и скорость: Оптимизирован для плохих сетевых условий и высокой потери пакетов, значительно превосходя классические аналоги (например, DNSTT).

- Низкие накладные расходы: Имеет собственный протокол с низким уровнем служебных данных.

- Балансировка: Поддерживает работу с несколькими резолверами (DNS-серверами) одновременно, а также автоматическое переключение при сбоях.

- Кроссплатформенность: Консольная версия работает практически на любой системе (Windows, Linux, Termux), а также существуют нативные клиенты для Android (например, MasterDnsVPN AndroidGG).

*Для использования системы необходимы два компонента:*

- Серверная часть: Разворачивается на личном VPS (виртуальном сервере) или в облачных функциях (включая скрипты для Docker).

- Клиентская часть: Подключается к вашему серверу с помощью специальных ключей и конфигураций.

**Прежде чем начать, прочитайте следующие инструкции:**

[Инструкция по выбору VPS и настройке домена.](vps.md)

[О DNS-резолверах и где их взять.](resolvers.md)

## 1. Настройка на сервере

Здесь рассматривается вариант, когда вы купили машину на Linux.

Подключитесь к машине по ssh, выполните скрипт для установки, который подготовил автор проекта:

```bash
bash <(curl -Ls https://raw.githubusercontent.com/masterking32/MasterDnsVPN/main/server_linux_install.sh)
```

Дождитесь, пока скрипт сделает свою работу и выведет в терминал следующие строки:

```bash
Configuration
Attention: Set your NS domain.
Enter your Domain (e.g. vpn.example.com):
```

Здесь мы указываем наш домен под NS записью (если воспользовались инструкцией по регистрации домена через Digiplate+Cloudflare, то в нашем случае это будет **v.example.dpdns.org**)

После этого скрипт сообщает нам:

- ключ шифрования

- команды по управлению сервисом (systemctl и journalctl)

Скопируйте ключ, он нам понадобится для настройки клиента. Если вы не знаете наизусть команды по управлению сервисами в Linux, сохраните их на будущее.

Сервер уже работает в фоне, можно приступать к настройке клиента.

**Не сообщайте третьим лицам пару домен+ключ шифрования и проверяйте конфиги/скриншоты на их наличие, если не хотите, чтобы к вашему серверу подключался кто-то, кроме вас/доверенных лиц!**

## 2. Настройка на клиенте

### 2.1 Windows/Linux

Скачайте билд для вашей системы со страницы [репозитория](https://github.com/masterking32/MasterDnsVPN/#section-21-use-prebuilt-releases--recommended) и распакуйте его.

Для начала, следует изменить некоторые параметры в файле **client_config.toml**:

`DOMAINS = ["v.domain.com"]`

Указываем наш домен, который мы прописали в скрипте на сервере.

`ENCRYPTION_KEY = "smoke-test-key-12345678901234567890123456789012"`

Вставляем ключ шифрования, который мы скопировали после установки.

На текущий момент этих настроек достаточно для тестов. Сохраните файл.

Откройте **client_resolvers.txt** и добавьте сюда публичные DNS-резолверы. 

Запустите исполняемый файл. ПО самостоятельно проверит резолверы на доступность и по окончанию проверки сообщит об этом в терминале:

![@](https://github.com/sinekdoxa/sinekdoxa.github.io/blob/main/docs/pics/mdnsvpnwinready.png?raw=true)

По умолчанию, открывается порт 18000 на локалхосте. Укажите адрес SOCKS5 прокси в удобном для вас расширении для браузера (FoxyProxy, например) и попробуйте выйти в интернет.

### 2.2 Android (MasterDnsVpn-GG)

[Github-репозиторий клиента](https://github.com/RevocGG/MasterDnsVPN-AndroidGG)

Скачайте билд для своей архитектуры и установите на телефон.

На главном экране нажмите кнопку **Add**.

Имя профиля может быть любым. **SOCKS5 / TUN** – на ваше усмотрение.

Нажмите на **Edit Resolver List**, вставьте список DNS резолверов, либо нажмите на голубую кнопку файла со стрелкой (справа сверху), чтобы импортировать резолверы из файла. После добавления нажмите галочку в правом верхнем углу.

Во вкладке **Tunnel Identity & Security** укажите свой домен (v.example.dpdns.org) и **ENCRYPTION_KEY**, который был ранее скопирован из терминала.

Сохраните конфиг, нажмите Start. Приложение просканирует DNS резолверы на доступность. 

По окончанию сканирования телефон разок завибрирует, будет указано кол-во доступных резолверов и локальный SOCKS5 localhost:18000. Если при конфигурации профиля выбран TUN, то можно сразу проверять работоспособность. 

Если вы выбрали SOCKS5 – пропишите IP:PORT в нужном приложении и проверьте работу через прокси.



### 2.3 Android (WhiteDNS)

[Github-репозиторий клиента](https://github.com/WhiteDNS/WhiteDNS-Android)

**WhiteDNS** опирается на проект MasterDNS Client и использует StormDNS — форк MasterDNS от [nullroute1970/StormDNS](https://github.com/nullroute1970/StormDNS).

*Возможности:*

- Android-клиент для DNS-туннелирования через WhiteDNS / StormDNS. 
  Режим прокси с поддержкой локального SOCKS5 и опциональным мостом HTTP-прокси. 

- Режим VPN с использованием Android VpnService и встроенных нативных библиотек tun2proxy. 

- Поддержка встроенных и пользовательских профилей серверов. 
  Функции импорта и экспорта профилей по ссылке `stormdns://`. 

- Управление профилями резолверов, включая проверку корректности и использование предустановленных настроек. 

- Настройки раздельного туннелирования (split tunneling) для маршрутизации VPN. 

- Отображение в реальном времени логов соединения, состояния резолвера, хода выполнения и статистики трафика. 

- Уведомления от службы, работающей в фоновом режиме, для длительных сеансов прокси и VPN. 

- Интерфейс на базе Jetpack Compose с использованием компонентов Material 3.

Несмотря на то, что приложение сделано для работы через форк [StormDNS](https://github.com/nullroute1970/StormDNS), ничто не мешает нам использовать его для подключения к MasterDnsVPN серверу.

**Из преимуществ** относительно MasterDnsVPN-GG приложение умеет устанавливать связь с Интернетом до окончания первоначальной проверки резолверов, что приводит нас к более быстрому подключению, а также имеет конфиги, заготовленные разработчиками. К преимуществам можно отнести встроенный сканер резолверов на доступность с последующим сохранением, чтобы не перепроверять все их при каждом запуске.

**К минусам** можно отнести меньшее количество стратегий балансировки, относительно MasterDnsVPN-GG, поскольку в StormDNS их всего 4 против 8 в MasterDnsVPN.

[Скачайте билд](https://github.com/WhiteDNS/WhiteDNS-Android/releases/latest) для вашей архитектуры и установите на смартфон.

Перейдите в раздел `Profiles` > `Connection` > `Create` > Дайте название серверу, укажите NS домен (v.example.dpdns.org) и ключ шифрования. **Метод шифрования должен совпадать на клиенте и на сервере** (по умочанию XOR).

Перейдите во вкладку `Resolvers` > `Create` > Вставьте заранее подготовленные резолверы, либо нажмите `Import File`.

Во вкладке `Setting` можно создать конфиг со своими значениями MTU, стратегией балансировки, дубликацией пакетов (раздельно для Upload/Download), сжатием и т.п. 

Перейдите в раздел `Connect`. По своему желанию выберите Proxy Mode, либо Full VPN.

Если вы до сих пор не подобрали параметры в конфиге для своего ISP, можно провести параллельный тест на конфигах, подготовленных разработчиками приложения. Включите `Parallel Test`, выберите **WhiteDNS configs**, **Agressive configs** и свои конфиги. 

По нажатию кнопки `Connect` приложение проверит резолверы, скорость и пинг на выбранных конфигурациях, определит лучший конфиг по скорости и выйдет в сеть с этими параметрами.

*Может случиться такая ситуация, что приложение успешно проверяет конфиги, но когда пытается подключиться через наилучший, не может проверить резолверы и соединение прерывается. В таком случае сохраните профиль, отключите `Parallel Test` и попробуйте соединиться снова, выбрав этот профиль. Баг это или особенность связки WhiteDNS+MasterDnsVPN - загадка.*

В разделе `Scan` Вы можете проверить заготовленные резолверы на доступность. Сделано на случай, если вы скачали публичный список [отсюда](resolvers.md#1-публичные-списки-для-самостоятельного-сканирования), и Вам нужно отсеять те, с которыми соединения точно не получится (например, вы проводите сканирование в условиях БС).

Нажмите `Select File`, найдите .txt файл с резолверами, выберите количество воркеров (чем больше, тем быстрее) и запустите сканирование. По окончанию проверки нажмите `Save As` и дайте название готовому списку. Затем, в разделе `Connect` в пункте `Resolver` выберите этот список.

### 2.4 iOS (zanoza)

[Github-репозиторий клеинта](https://github.com/palmbeachpete9/masterdns-zanoza-ios)

iOS-клиент для MasterDnsVPN — VPN на основе туннелирования через DNS для сетей с жёсткой цензурой.

Приложение использует оригинальное ядро Go-клиента MasterDnsVPN, поднимает локальный SOCKS5-прокси на `127.0.0.1:41080` для подключения через сторонние VPN-приложения (Shadowrocket, Happ и т.п.), которые уже пропускают весь системный трафик через DNS-туннель. Собственный VPN-профиль Zanoza создаёт только в том случае, если Вы подписали приложение действительным сертификатом Apple.

1. Запустите Zanoza и нажмите Импорт.

2. Введите делегированный домен с вашего сервера MasterDnsVPN (то же значение, что в NS-записи, `v.domain.ru`).

3. Введите ключ шифрования, полученный при настройке сервера.

4. Нажмите Импорт, затем кнопку "питания" для запуска туннеля. В настройках профиля Метод шифрования должен совпадать со значением `DATA_ENCRYPTION_METHOD` в `server_config.toml` сервера (совпадает по умолчанию, XOR).

5. SOCKS5-прокси запустится на `127.0.0.1:41080`. Откройте Shadowrocket / Happ / и т.п., добавьте SOCKS5-прокси на этот адрес, и подключитесь.

**IPA-файл не подписан.** Подпишите и установите его на устройство одним из способов:

- [Sideloadly](https://sideloadly.io/) — перетащите IPA в окно, подпишите своим Apple ID и установите через USB.

- AltStore / SideStore — установка прямо на устройстве, после первой настройки Mac уже не нужен.

Перед первой установкой включите на iPhone **Настройки → Конфиденциальность и безопасность → Режим разработчика.**

## 3. Тонкая настройка клиента

*Перед выполнением нижеописанных пунктов сделайте бэкап конфигурационного файла.*

*Инструкция писалась с упором на десктопные клиенты. При использовании на мобильных устройствах суть не меняется, параметры как правило остаются те же. Редактирование происходит не в конфигурационном файле, а в GUI клиента.*

**Я не хочу сканировать все резолверы каждый раз при запуске**

Отредактируйте следующие переменные в конфигурационном файле:

```bash
SAVE_MTU_SERVERS_TO_FILE = true
MTU_SERVERS_FILE_FORMAT = "{IP}"
```

Просканируйте резолверы, откройте создавшийся .log файл, скопируйте IP адреса "живых" DNS резолверов. Сохраните их либо в client_resolvers.txt, либо создайте новый файл, вставьте адреса туда и запускайте исполняемый файл с параметром:

```bash
MasterDnsVPN.exe -resolvers <filename>
```

**Скорость слишком низкая/при скачивании файла происходит "затуп" и загрузка встает на некоторое время**

Найдите строку в терминале после сканирования:

```bash
2026/05/23 23:42:14 [MasterDnsVPN Client] [INFO] Global MTU Configuration -> Upload: 136, Download: 2929
```

Попробуйте ограничить Download MTU. Для этого в конфигурационном файле отредактируйте параметр:

```bash
MAX_DOWNLOAD_MTU = 3000
```

Измените значение до трехзначного (например, 800) либо до четырехзначного (например, 1500), перезапустите клиент и проверьте стабильность скачивания.

Может сложиться такая ситуация:

```bash
[MasterDnsVPN Client] [INFO] Global MTU Configuration -> Upload: 136, Download: 233
```

Прокрутите терминал выше. Клиент выводит списком IP адреса DNS резолверов и подобранное для них MTU. Возможно, встроенный балансир ограничил общее MTU по самому низкому значению вместо того, чтобы исключить резолвер. 

Помогите ему, отредактировав следующий параметр в конфиге:

```bash
MIN_DOWNLOAD_MTU = 150
```

Укажите значение повыше, чтобы резолвер с низким MTU был отсеян при первоначальной проверке. Перезапустите клиент и проверьте Global MTU после перескана.

**Дополнительные возможности поиграться с конфигом**

Вы можете включить сжатие:

```bash
# ------------------------------------------------------------------------------
# 5) Compression
# What this section does:
# - Controls payload compression negotiation for upload and download directions.
# ------------------------------------------------------------------------------

# Allowed values:
# 0 = OFF
# 1 = ZSTD
# 2 = LZ4
# 3 = ZLIB
UPLOAD_COMPRESSION_TYPE = 0
DOWNLOAD_COMPRESSION_TYPE = 0
```

Измените стратегию для баланса резолверов:

```bash
# Resolver balancing strategy.
# Allowed values:
# 1 = Random
# 2 = Round Robin
# 3 = Least Loss
# 4 = Lowest Latency
# 5 = Hybrid Score (loss-first + latency-aware)
# 6 = Loss Then Latency (loss shortlist, then latency, then rotate among near-top)
# 7 = Least Loss Top Random (random choice inside the best 10% loss tier)
# 8 = Least Loss Top Round Robin (round-robin inside the best 10% loss tier)
# Modes 3, 4, 5, 6, 7, and 8 use runtime feedback from sends/successes.
RESOLVER_BALANCING_STRATEGY = 3
```

Совет от автора: попробуйте Hybrid Score. Изначально соединение может быть меделнным, но затем софт делает баланс и скорость возрастает. Однако стоит перебрать все стратегии и найти лучшую для своих резолверов.

Увеличьте число повторяющихся пакетов:

```bash
# Duplicate each normal outgoing tunnel packet this many times.
# Clamped in code to [1, 10].
# Higher values improve resilience but increase bandwidth and CPU usage.
PACKET_DUPLICATION_COUNT = 2
```

Значение выше 1 повышает стабильность соединения. 

Стоит учитывать, что потребление трафика тем выше, чем большее число указано. Если у вас ограниченние по трафику от ОПСОСа, например, то можно случайно потратить весь пакет. 

*Примечание: существует лимит от сервера на это значение.*

**1 Mbit/s is fine too, но можно еще быстрее?**

Можно. Обратитесь к странице: [Проверка резолверов на скорость](speedtest.md)
