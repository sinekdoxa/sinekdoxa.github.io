# **Этот сайт содержит инструкции по настройке технологии VPN over DNS.**

**VPN over DNS (DNS-туннель)**— это технология инкапсуляции интернет-трафика в стандартные DNS-запросы и ответы. Она используется для обхода строгих сетевых ограничений (например, в сетях с платным Wi-Fi или фаерволами), так как DNS-трафик редко блокируется.
Для понимания принципа работы, преимуществ и недостатков можно выделить следующие ключевые аспекты:

**Как это работает**

1. **Преобразование данных:** Ваш клиент маскирует обычный интернет-трафик (веб-страницы, сообщения) под текстовые DNS-запросы (например, `://example.com`).

2. **Маршрутизация:** Запросы проходят через стандартный DNS-сервер вашего провайдера к удаленному серверу-шлюзу (DNS-серверу хакера или сервиса).

3. **Распаковка:** Шлюз извлекает из запросов реальный трафик, отправляет его в сеть, получает ответ и возвращает его на ваше устройство в виде замаскированного DNS-ответа.

**Плюсы и Минусы**

- **Плюсы:** Работает даже там, где заблокирован стандартный доступ в интернет или VPN-порты.

- **Минусы:** Крайне низкая скорость. Размер данных, которые можно передать в одном DNS-запросе, сильно ограничен, поэтому VPN over DNS не подходит для стриминга или загрузки больших файлов.

**Где применяется**

- **Обход цензуры и блокировок:** Используется в качестве крайней меры для получения доступа к сети в условиях жестких ограничений или при отсутствии денег на балансе в отельном/публичном Wi-Fi.

- **Тестирование безопасности:** Применяется системными администраторами для проверки уязвимостей сетей, а злоумышленниками — для скрытой передачи данных (data exfiltration).

- **Популярные инструменты:** Для реализации этого метода часто используют такие утилиты, как DNSTT (написан на Go) или мобильные приложения вроде SlowDNS.

*Рассматривайте метод VPN over DNS только в качестве аварийного канала связи!*
